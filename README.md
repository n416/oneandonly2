# OneAndOnly2

このプロジェクトは、**Tauri + React + TypeScript** によるフロントエンド・デスクトップ基盤と、**Python (FastAPI + llama-cpp/transformers)** によるローカルLLMサーバー（サイドカー）を組み合わせたAI駆動型TRPGアプリケーションです。
前作「OneAndOnly1 (Vanilla JS + Gemini API)」の完全互換を目指しつつ、クラウドAPIに依存しないスタンドアロン構成へと進化しました。

## プロジェクト構成

- `/src` - React（フロントエンド）のソースコード
- `/src-tauri` - Tauri（Rustバックエンド）のソースコードと設定
- `/llm-api` - PythonベースのローカルLLM APIサーバー（Git Submodule）
- `/scripts` - パッケージング等の自動化スクリプト

---

## コア機能と特徴

- **AIによるシナリオ進行**: ローカルLLMがゲームマスター（GM）となり、プレイヤーの行動に対して柔軟にシーンを描写します。
- **選択肢の自動生成（お助け機能）**: どう行動していいか悩んだ時、プレイヤーの「所持アイテム」を考慮した最適な行動候補をAIが提案します。
- **AIガチャ＆キャラクター図鑑**: カスタムプロンプトからキャラクターやアイテムをAIが生成。レア度（★1〜★5）も自動判定して図鑑（倉庫）に保存されます。
- **完全ローカル動作**: GGUFモデルを使用し、インターネットに接続せずとも強力なAI TRPG体験が可能です。

## 必須設定： `.env` ファイルの準備

プロジェクトを動作させるには、プロジェクトルート階層に `.env` ファイルを作成し、AIモデルのパスや設定を記述する必要があります。

```env
# ★★★ GGUFモデルのパスを指定 (必須) ★★★
MODEL_PATH=./models/ibm/test.gguf

# --- Optional Llama.cpp Settings ---
# GPUレイヤー数（VRAMに載せる数。GPUがない場合は 0 にする）
N_GPU_LAYERS=0
# コンテキストサイズ（トークンの記憶容量。長文シナリオに対応するために大きめに設定）
N_CTX=65535
```
※ 使用するモデル（`.gguf`）は各自でダウンロードし、`models/` ディレクトリ等に配置してください。

## 開発・デバッグ (Development & Debugging)

開発中は、**プロジェクトルート（一番上の階層）** から一切移動せずにすべてのコマンドを実行できます。

### 1. Python APIサーバーの起動（手動デバッグモード）
```powershell
npm run api:dev
```
裏側で自動的に `llm-api` ディレクトリへ移動し、`python main.py` を実行します。
AIモデルの読み込み状況やAPIリクエストのログが画面に直接表示されるため、Python側のデバッグはこちらを見ながら行います。

### 2. フロントエンドの起動
UIの調整やデバッグを行いたい場合、以下のいずれかの方法で起動します。

**A. Tauri デスクトップアプリとして起動**
```powershell
npm run tauri dev
```
Tauriのウィンドウが立ち上がります。（※起動時にサイドカーが見つからない場合は自動的に手動デバッグモードと見なされ、上記1で立ち上げたサーバーと通信します）

**B. Webブラウザでデバッグ起動（推奨）**
```powershell
npm run dev
```
`http://localhost:1420` にアクセスし、Chrome開発者ツール等で高度なUI/Stateデバッグが可能です。

## Python API (llm-api) のビルド手順

Tauriの実行ファイル（exe）からPythonサーバーを自動起動（サイドカー）させるため、あらかじめPython側を単体実行可能な `.exe` にコンパイルしておく必要があります。

```powershell
cd llm-api
pip install pyinstaller
pyinstaller --onefile main.py
```

> **Note**: ビルドによって生成された `dist/main.exe` は、手動で移動する必要はありません。後述のパッケージ化スクリプトが自動的に適切な場所へコピー・リネームして組み込みます。

---

## 配布用パッケージの自動構築 (Packaging)

アプリを他のユーザーに配布したり、完成品としてスタンドアロン実行するための最終構成（exe本体 ＋ 設定ファイル ＋ モデルファイル）を自動生成します。

```powershell
npm run package
```

### スクリプトの自動処理内容
1. `llm-api/dist/main.exe` を Tauriのサイドカー用ディレクトリ（`src-tauri/bin/`）にコピー・リネーム。
2. Tauri アプリケーション全体のリリースビルドを実行。
3. プロジェクト直下に `dist_package/` フォルダを作成し、以下の配布構成に集約。

### 最終的な配布用フォルダ構成 (`dist_package`)
```text
dist_package/
 ├── tauri-app.exe   (← ユーザーが実行するアプリ本体)
 ├── .env            (← LLMのモデルパスなどの設定。ユーザーが書き換え可能)
 └── models/         (← GGUFなどの重いモデルファイルを置くフォルダ)
```
この構成により、ユーザーはPython環境をインストールすることなく、`.exe` を起動するだけで裏側でローカルLLMサーバーが立ち上がり、すぐにチャットを利用できます。モデルの差し替えや `N_CTX` の調整も配布後に `.env` を書き換えるだけで対応可能です。
